What is Server-Side Web Scripting?

Server - Side Web Scripts basically run on a server. 
It is used to interact with the back-end structure of a web application such as
accessing data or information from a database stored in a server so as to facilitate data transfer 
or produce dynamic content to a web resource. 

Why Use Server Side Scripting?

It enables you to produce dynamic web content 

It enables you to customize response for the user

It enables you to gather and retrieve data from the database

It is used to create large web application such as Bank Apps

Java 

Java can also be used for web applications. Java web applications enable an individual to create dynamic web pages through SERVLETS or JavaServer Pages a.k.a JSP and also incorporating static web resources like HTML pages and styles. 

These web applications can be deployed by archiving them as WAR files. A WAR file or Web Archive File is a complete archive of web application content.

Java Web Standard Technologies

Servlets
Servlets are Java Clases which extends "HttpServlet" which allows a user to create dynamic and interactive web pages through HTTP request and HTTP response .

Java Server Pages
JavaServer Pages (JSP) are files which holds combined HTML and Java code. JSP is basically Java code directly embeded into HTML pages. Having this kind of setup makes it way easier to create dynamic pages compared to servlets.  JSP is actually compiled into a servlet. 

LIFE CYCLE

INITIALIZATION

First, init() method is called to initialize the servlet.
The init() Method
The init method is used only once in the entire servlet life cycle. It is called during the creation of the servlet and it is only used for one-time initializations. During a servlet invocation, one instance of a servlet is created wherein every request turns to a new thread which is then handed of the necessary doXXX(). This method basically initializes the data that will be used in the entire service life.

public void init() throws ServletException { // Initialization code... }

SERVICE

Second, service() method is then called upon to process client request.
The service() Method
The service() method is the main method which performs the actual task. Whenever a request for a servlet is made a new thread is created and calls a service. The method checks the HTTP request type and calls the appropriate doXXX(), such as;

  • GET for doGet()
  • POST for doPost()
  • PUT for doPut()
  • DELETE for doDelete()

public void service(ServletRequest request, ServletResponse response) throws ServletException, IOException { }

DESTROY

Finally, destroy() method is called to terminate the servlet. The destroy() Method The destroy() method is also called only once, like init(), but this time during the end of the servlet's cycle. This method enables the servlet to close database connections, stop background threads, and perform other cleanup activities.

public void destroy() { // Finalization code... }

JSP COMPILATION


The JSP engine compiles a JSP if it has never been compiled before or if there are any modifications done recently.
The compilation has three main steps;

  1. Parse the JSP
  2. Convert the JSP to servlet
  3. Compile the servlet

JSP INITIALIZATION

jspInit() is the equivalent of init() method in servlet.
Initialization in JSP is basically the same with servlets wherein connections are established and files are prepared.

JSP EXECUTION

_jspService() method is invoked to handle client requests just like service() method in servlets. It takes both HttpServletRequest and HttpServletResponse as parameters to serve appropriate responses for each request.

JSP CLEANUP

This is where the destruction phase of the JSP cycle comes into place. Cleaup is the equivalent to servlet's destroy phase and is invoked using the method jspDestroy(). This phase includes closing all the connections made in the initialization and closing the files.

PHP

PHP was initially stood for Personal Home Page,  however it was then changed to a recursive acronym that means PHP: Hypertext Preprocessor.

PHP is a widely-used open source general-purpose scripting language that is especially suited for web development and can be embedded into HTML.

In 1994, PHP was intially created and designed by Rasnus Lerdorf and now being implemented by Zend Technologies with Zend Engine.


PHP Scripts uses Three main areas

  • Server-side scripting

    • Known as the main objective and the most traditional of all the PHP Scripts. PHP parser (CGI or server module), a Web Server and a Web Browser is required in order to execute and run PHP. Web Server because it is needed together with a PHP installation which is connection. Also, you will have the ability to access the PHP program output with a web browser which you can view the PHP page through the Web Server.

  • Command line scripting

    • PHP Script can also generate and execute without using any server or browser but only required a PHP parser to imply the idea for scripts often executed using CRON/Task Scheduler. It can also be utilized for any text processing tasks.

  • Writing desktop applications

    • PHP is not usually the typical language to use in making and generating a desktop application with a GUI. However, you can still able to use it for the client-side applications if you want to be advanced.
